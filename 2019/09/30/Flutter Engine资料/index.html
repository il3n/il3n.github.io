  2. flutter’s modes

    • 1). Debug used by flutter run build with sky/tools/gn --android or sky/tools/gn --ios
    • 2). Release used by flutter run --release build with sky/tools/gn --android --runtime-mode=release or sky/tools/gn --ios --runtime-mode=release
    • 3). Profile used by flutter run --profile build with sky/tools/gn --android --runtime-mode=profile or sky/tools/gn --ios --runtime-mode=profile
    • 4). Headless test used by flutter test build with sky/tools/gn

  3. Compile the engine (在src目录下执行下面的步骤)

    • 1). git pull upstream master in src/flutter to update the Flutter Engine repo.
    • 2). gclient sync to update dependencies.
    • 3). Prepare build files
      • ./flutter/tools/gn --ios --unoptimized to prepare build files for device-side executables (or --ios --simulator --unoptimized for simulator).
      • This also produces an Xcode project for working with the engine source code at out/ios_debug_unopt
    • 4). ./flutter/tools/gn --unoptimized to propare the build files for host-side executables.
    • 5). ‘ninja -C out/ios_debug_unopt && ninja -C out/host_debug_unopt’ to build all artifacts(use out/ios_debug_sim_unopt for Simulator).

  4. 最佳实践

    • 1). 配置depot_tools工具
      git clone https://chromium.googlesource.com/chromium/tools/depot_tools.git，然后将depot_tools加入环境变量export PATH="$PATH:~/Flutter/depot_tools"

    • 2). 本地创建名为engine的文件夹，并在engine内创建.gclient文件，内容如下，将url改为自己的engine地址

    • 3). 在engine目录内，执行gclient sync，获取flutter依赖的所有源码，此处比较耗时。

      • 这一步很容易出现各种网络导致的错误。一般的解决办法是使用代理工具，并在.bash_profile中添加export http_proxy=http://127.0.0.1:1087;和export https_proxy=http://127.0.0.1:1087，然后source ~/.bash_profile，但是要记得代码同步完成以后要删除。更多网络方面的设置，可以参考Flutter Engine 编译指北

    • 4). 同步代码 cd ~/Flutter/engine/src/flutter，git remote add upstream https://github.com/flutter/engine.git，git pull upstream master

    • 5). 同步版本。

      • 执行cat ~/Flutter/flutter/bin/internal/engine.version可以查看当前使用的flutter的engine的commit-id。
      • 执行cd ~/Flutter/engine/src/flutter，切换到engine/src/flutter目录下。执行git reset --hard commit-id，回滚代码。
      • 查看当前版本git rev-parse HEAD
      • 执行gclient sync --with_branch_heads --with_tags，再次同步依赖的代码。

    • 6). 编译

      • 在engine/src目录下执行./flutter/tools/gn --ios --unoptimized && ./flutter/tools/gn --unoptimized。生成iOS未优化的debug版本工程。
      • 执行ninja -C out/ios_debug_unopt && ninja -C out/host_debug_unopt，编译工程。
      • 或者使用./flutter/tools/gn --ios --unoptimized --runtime-mode=release && ./flutter/tools/gn --unoptimized --runtime-mode=release，生成arm64未优化的release版本。
      • 执行ninja -C out/ios_release_unopt && ninja -C out/host_release_unopt，编译工程。

    • 7). 使用

      • 在flutter工程中，执行flutter build ios --debug --local-engine-src-path ~/Flutter/engine/src/ --local-engine=ios_debug_unopt，使用本地engine编译工程。

    • 8). 运行

      • 使用xcode编译运行，即可使用本地的engine。

  5. 常见问题

    • 1). 一般在gclient sync期间出现的问题，都是网络导致的，解决的办法是使用网络代理，并在.bash_profile中添加http_proxy和https_proxy。请参考Flutter Engine 编译指北
